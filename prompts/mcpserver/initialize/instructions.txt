# contract: v1.0.1

Evidra is a reliability recorder for infrastructure automation.
It measures safe execution behavior. It does not block operations.

Protocol for every infrastructure mutation:
1) call `prescribe` BEFORE execution
2) call `report` AFTER execution (success or failure)

Mutation examples (must use prescribe/report):
- kubectl apply/delete/patch/create/replace/rollout restart
- helm install/upgrade/uninstall/rollback
- terraform apply/destroy/import

Read-only examples (no prescribe/report):
- kubectl get/describe/logs/top/events
- helm list/status/template
- terraform plan/show/output

Critical invariants:
- Do not execute mutate commands until prescribe returns ok=true with prescription_id.
- Every prescribe must have exactly one report.
- Always include actor.skill_version (set to this contract version).

Rules:
- Report failures with non-zero exit_code.
- On retry, call prescribe again for each attempt.
- If unsure whether command mutates state, call prescribe.
- Do not use prescribe/report for non-infrastructure tasks (docs, coding, analysis).

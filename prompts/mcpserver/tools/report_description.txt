# contract: v1.0.1

Record outcome AFTER an infrastructure operation completes.

Required inputs:
- prescription_id (from prescribe)
- exit_code (0 success, non-zero failure)

Optional inputs:
- actor (type, id, origin; should match prescribe actor)
- actor.version, actor.skill_version
- artifact_digest (actual applied artifact digest)
- session_id, operation_id, span_id, parent_span_id

Terminal outcome rule:
- Every prescribe must end with exactly one report, including failed/aborted attempts.
- Retries require a new prescribe/report pair for each attempt.

Rules:
1) Always report failures; do not hide non-zero exit codes.
2) Do not report twice for the same prescription_id.
3) Do not report another actor's prescription_id.
4) If prescription_id is lost, call prescribe again before execution.
5) Actor identity should match the original prescribe actor.
6) Include actor.skill_version for behavior slicing.

Returns:
- ok
- report_id
- signals (optional)
- error (if ok=false)
